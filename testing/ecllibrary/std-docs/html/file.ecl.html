FUNCTION
INTEGER4 COMPAREFILES (varstring lfn1, varstring lfn2, boolean logical_only=TRUE, boolean use_crcs=FALSE)
Tree
Compares two files, and returns a result indicating how well they match.

Parameters
file1 The logical name of the first file.
file2 The logical name of the second file.
logical_only Whether to only compare logical information in the system datastore (Dali), and ignore physical information on disk. [Default TRUE]
use_crcs Whether to compare physical CRCs of all the parts on disk. This may be slow on large files. Defaults to FALSE.
Returns
0 if file1 and file2 match exactly 1 if file1 and file2 contents match, but file1 is newer than file2 -1 if file1 and file2 contents match, but file2 is newer than file1 2 if file1 and file2 contents do not match and file1 is newer than file2 -2 if file1 and file2 contents do not match and file2 is newer than file1

FUNCTION
ADDFILERELATIONSHIP (varstring primary, varstring secondary, varstring primaryflds, varstring secondaryflds, varstring kind='link', varstring cardinality, boolean payload, varstring description='')
Tree
Defines the relationship between two files. These may be DATASETs or INDEXes. Each record in the primary file should be uniquely defined by the primaryfields (ideally), preferably efficiently. This information is used by the roxie browser to link files together.

Parameters
primary The logical filename of the primary file.
secondary The logical filename of the secondary file.
primaryfields The name of the primary key field for the primary file. The value "__fileposition__" indicates the secondary is an INDEX that must use FETCH to access non-keyed fields.
secondaryfields The name of the foreign key field relating to the primary file.
relationship The type of relationship between the primary and secondary files. Containing either 'link' or 'view'. Default is "link".
cardinality The cardinality of the relationship. The format is :. Valid values are "1" or "M".
payload Indicates whether the primary or secondary are payload INDEXes.
description The description of the relationship.

FUNCTION
varstring FSPRAYFIXED (varstring sourceIP, varstring sourcePath, integer4 recordSize, varstring destinationGroup, varstring destinationLogicalName, integer4 timeOut=-1, varstring espServerIpPort=GETENV('ws_fs_server'), integer4 maxConnections=-1, boolean allowOverwrite=FALSE, boolean replicate=FALSE, boolean compress=FALSE, boolean failIfNoSourceFile=FALSE, integer4 expireDays=-1)
Tree
Sprays a file of fixed length records from a single machine and distributes it across the nodes of the destination group.

Parameters
sourceIP The IP address of the file.
sourcePath The path and name of the file.
recordsize The size (in bytes) of the records in the file.
destinationGroup The name of the group to distribute the file across.
destinationLogicalName The logical name of the file to create.
timeOut The time in ms to wait for the operation to complete. A value of 0 causes the call to return immediately. Defaults to no timeout (-1).
espServerIpPort The url of the ESP file copying service. Defaults to the value of ws_fs_server in the environment.
maxConnections The maximum number of target nodes to write to concurrently. Defaults to 1.
allowOverwrite Is it valid to overwrite an existing file of the same name? Defaults to FALSE
replicate Whether to replicate the new file. Defaults to FALSE.
compress Whether to compress the new file. Defaults to FALSE.
failIfNoSourceFile If TRUE it causes a missing source file to trigger a failure. Defaults to FALSE.
expireDays Number of days to auto-remove file. Default is -1, not expire.
Returns
The DFU workunit id for the job.

Parameters
sourceIP The IP address of the file.
sourcePath The path and name of the file.
sourceCsvSeparate The character sequence which separates fields in the file.
sourceCsvTerminate The character sequence which separates records in the file.
sourceCsvQuote A string which can be used to delimit fields in the file.
sourceMaxRecordSize The maximum size (in bytes) of the records in the file.
destinationGroup The name of the group to distribute the file across.
destinationLogicalName The logical name of the file to create.
timeOut The time in ms to wait for the operation to complete. A value of 0 causes the call to return immediately. Defaults to no timeout (-1).

maxConnections The maximum number of target nodes to write to concurrently. Defaults to 1.
allowOverwrite Is it valid to overwrite an existing file of the same name? Defaults to FALSE
replicate Whether to replicate the new file. Defaults to FALSE.
compress Whether to compress the new file. Defaults to FALSE.
sourceCsvEscape A character that is used to escape quote characters. Defaults to none.
failIfNoSourceFile If TRUE it causes a missing source file to trigger a failure. Defaults to FALSE.
recordStructurePresent If TRUE derives the record structure from the header of the file.
quotedTerminator Can the terminator character be included in a quoted field. Defaults to TRUE. If FALSE it allows quicker partitioning of the file (avoiding a complete file scan).
expireDays Number of days to auto-remove file. Default is -1, not expire.

The DFU workunit id for the job.
